02変更をチームに送るまで

GitHub Desktopだけで完結します。ターミナルもコマンドも使いません。

  1. まず最新を取り込む — 作業を始める前に、GitHub Desktop左上の Fetch origin → Pull origin を押す。 これをサボると、自分の変更と誰かの変更がぶつかる(コンフリクト)原因になります。作業を始める前に必ず1回。
  2. 自分の作業ブランチを作る／切り替える — 左上の Current Branch から New Branch。名前は feature/やること名 のように。 mainブランチのままでは絶対に作業しない（後述のFAQ参照）。
  3. ファイルを直す — Claudeやエディタでコードを書く／直す。
  4. 保存する(コミット) — GitHub Desktop左下に、変更した内容とメッセージ欄が出るので、 「何をしたか」を日本語で一言書いて Commit to feature/〜 を押す。 例: 「カードの選択画面を直した」「人数選択を6人までにした」
  5. チームに送る(プッシュ) — 上の Push origin を押す。これで自分のPCの変更がGitHubに届く。
  6. Pull Requestを作る — GitHub Desktopに出てくる Create Pull Request を押すとブラウザが開くので、 何をしたか簡単に書いて作成。 「できました」を口頭やDiscordで言うだけで終わらせず、必ずPRを作ってください。これが無いと、あなたの変更は本番に反映されません。

03よくある詰まりポイント

タップすると開きます。

「大富豪　第◯弾」ってフォルダ、あれ何？触っていい？
⚠️ 触らないでください（過去に3回、これが原因で機能が反映されない事故が起きています）。
「大富豪　第七弾」「大富豪　第九弾」のような番号付きフォルダは、過去の記録用（アーカイブ）で、 ゲームの動作には一切使われていません。

実際にゲームが読み込んでいるのは、リポジトリの一番上の階層にある card.js と game.js の2つだけです。 ロジックを直したい時は、必ずこの2つを直接編集してください。
どのファイルを触ればいいか分からない

基本的にこの3つだけで、ほぼ全部作れます。

index.html … 画面・見た目・ボタンの動き(UI)全部
card.js … カード1枚1枚の強さ・役の判定
game.js … ゲーム全体の進み方(誰の番か、場に何が出てるか)

mainブランチを直接触っちゃダメなの？

はい、ダメです。mainは「今動いている本番」そのものなので、 直接壊れた変更が入ると全員が困ります。必ず自分のfeature/〜ブランチで作業して、 Pull Requestを経由してからmainに入るようにしてください。

Pushしようとしたらエラーが出た／コンフリクトと言われた

自分の変更と、他の人が先に送った変更が同じ場所を直していると起きます。 無理に一人で解決しようとせず、そのままの状態でDiscordに相談してください。 スクリーンショットがあると早く解決します。

自分の変更がサイトに反映されない

Pull Requestを作っただけではまだ本番に反映されません。誰かがマージ（mainへの取り込み） するまで待つ必要があります。マージされたら数分でVercelが自動で公開してくれます。
