# Contributing Guide

このリポジトリでの開発ルールです。全員必ず読んでください。

## ブランチ運用ルール

### ① `main` ブランチは触らない
- `main` は常に動く状態を維持します。
- **`main` に直接pushしない**（Branch protection ruleで禁止されています）。

### ② 作業ごとにブランチを作る
自分の作業内容に応じて、以下の命名規則でブランチを作成してください。

- `feature/xxx` … 新機能の追加（例: `feature/game-logic`, `feature/lobby`）
- `fix/xxx` … 不具合修正（例: `fix/card-render-bug`）

```bash
git checkout main
git pull origin main
git checkout -b feature/自分の担当機能名
```

### ③ 完成したらPull Requestを作成
「できました！」と口頭やDiscordで言うのではなく、必ずPull Requestを作成してください。

流れ：
1. 自分のブランチをpush
2. GitHub上でPull Requestを作成（何をしたか簡単に書く）
3. 他のメンバーにレビュー依頼
4. レビューOKならマージ

### ④ レビューについて
- レビューは最低1人以上必須です（Branch protection ruleで強制されています）。
- レビューでは「動くかどうか」だけでなく、「他の人が読んでも分かるか」も見てください。
- 分からないところは遠慮なく質問してOKです。学びの場でもあります。

## コミットメッセージの目安
細かいルールは設けませんが、何をしたか一目で分かるように書いてください。

例：
```
add: 手札からカードを出す処理を実装
fix: 革命後に強さの判定が逆になっていたのを修正
```

## 困ったとき
- マージ時にコンフリクトが起きたら、無理に一人で解決しようとせず、Discordで相談してください。
- わからないことがあれば遠慮なく聞いてください。「遊び心を忘れない」精神で楽しく作りましょう！
